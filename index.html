Es software libre

No solo es software libre, sino que ha hecho posible una explosión del desarrollo de software libre, sobre todo a partir del uso de GitHub como sitio estándar para depositar el código, y donde las fuentes están abiertas por omisión (al menos hasta hace poco). Software libre es el que tiene una licencia libre, que permite copiar, modificar y distribuir. Para descargarlo, usa los repositorios de tu distribución; puedes descargarte también una herramienta gráfica que incluye una línea de órdenes en windows.github.com o mac.github.com.

Clona el repo

git clone https://github.com/nick/nombre-repo
Esto crea una copia, un fork del repo. Los cambios a partir de ahora serán cambios locales. Necesitas estar autorizado para leer del repo, pero todos los repos tienen habilitada la lectura en la versión https, así que esto se puede hacer en cualquier caso..

¡Has creado un fork!

Crea forks en GitHub

Puedes crearlos también en el propio sitio, lo que facilita hacer una serie de cosas, como pedir que incorporen los cambios que se han hecho al original. Podéis hacerlo con este repo, por ejemplo. Lo veremos más abajo

Una vez clonado...

cd nombre-repo
git status
ls -a
Este último nos mostrará los ficheros bajados, pero también un fichero .gitignore oculto, y un directorio git oculto que es donde se encuentran efectivamente

O haz un fork de otro en GitHub

git clone git@github.com:minick/git-intro.git git remote add upstream https://github.com/JJ/git-intro.git
Con este último te aseguras de mantenerte al día de los últimos cambios al repo. Conviene siempre que actualices desde upstream antes de subir tus actualizaciones, para evitar conflictos.

Usa Oh-My-Zsh

O Bash-it o Oh-my-Fish

Es bastante conveniente que se use Linux en alguna de sus distribuciones para desarrollar. Se puede usar Windows, pero muchas de estas cosas no van a ser ni fáciles ni convenientes. De hecho, los GUIs de git te instalan una línea de órdenes similar a Linux para trabajar, pero de hecho la instalación de dependencias y todo tipo de bibliotecas es mucho más sencilla en Linux.

Nuevos ficheros

git add nombre-del-fichero.md
git add .
Es mejor añadir sólo los ficheros con los que se va a trabajar. Nunca se debe usar git add . ni git add * porque te puedes encontrar con sorpresas desagradables o simplemente ficheros que no deberían estar en un repo.
